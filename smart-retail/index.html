Core Features

Built from the ground up, this system provides a full suite of features for both customers and administrators.

Online Storefront

A public-facing interface where customers can browse products, view details, and add items to their shopping cart.

Secure Authentication

Robust customer registration and login system using password hashing (PASSWORD_DEFAULT) to ensure data security.

Dynamic Shopping Cart

A session-based shopping cart that allows users to manage their selections before proceeding to checkout.

Admin Dashboard

A secure backend portal for store managers to get a quick overview of recent orders and low-stock alerts.

Inventory Management

Administrators can track and update stock levels in real-time. The system automatically flags items running low.

Order Processing

Admins can view all customer orders, update their status (e.g., Pending, Shipped, Completed), and manage the fulfillment workflow.

Validation and Testing

The system's functionality was rigorously validated through a scenario-based practical approach to ensure reliability and bug-free operation.

Real-World Use Case Scenarios Tested:

  • New Customer Registration: A user successfully creates an account with unique email validation and secure password hashing.
  • Product Discovery & Selection: A customer browses the product listings, views a detailed product page, and adds an item to the cart.
  • Shopping Cart Management: A customer adds multiple items, updates quantities, and removes an item from the cart.
  • Order Processing: A logged-in customer proceeds to checkout, which validates stock levels before successfully placing an order.
  • Stock Deduction: On successful order placement, inventory levels for the purchased products are automatically deducted.
  • Payment Simulation: The system correctly simulates a payment transaction, recording its status in the database.
  • Admin Order Management: A store manager logs in, views the new order, and updates its status from 'Pending' to 'Shipped'.
  • Admin Reporting: The system correctly generates sales and low-inventory reports based on a set date range.

Comprehensive testing was performed at each stage to identify and resolve bugs, ensuring the system meets the needs of both customers and store managers.
